# Deployment Guide (Render)

Zero Hunger deploys as **two Render services** defined in [`render.yaml`](render.yaml):

| Service | Type | Folder | What it runs |
|---|---|---|---|
| `zero-hunger-api` | Web Service (Node) | `Backend/` | Express REST API + Socket.IO |
| `zero-hunger-web` | Static Site | `Frontend/` | Vite-built React SPA |

## Prerequisites

1. A **MongoDB** database (e.g. MongoDB Atlas free tier). Copy its connection string.
2. A **Gmail App Password** (Google account → Security → App passwords) for transactional email.
3. An **ImgBB API key** (https://api.imgbb.com/) for donation image uploads.
4. This repo connected to your Render account.

> ⚠️ Rotate any previously-committed secrets first — see [SECURITY.md](SECURITY.md).

## One-time deploy with the Blueprint

1. In Render, choose **New → Blueprint** and select this repository. Render reads
   `render.yaml` and creates both services.
2. Set the environment variables Render marks as required (they are `sync: false`,
   so they are never stored in git):

   **`zero-hunger-api`**
   | Variable | Value |
   |---|---|
   | `MONGO_URI` | your MongoDB connection string |
   | `EMAIL_USER` | your Gmail address |
   | `EMAIL_PASS` | your Gmail App Password |
   | `IMGBB_KEY` | your ImgBB key |
   | `FRONTEND_URL` | the web service URL (e.g. `https://zero-hunger-web.onrender.com`) |
   | `CORS_ORIGINS` | same as `FRONTEND_URL` (comma-separate if multiple) |
   | `JWT_SECRET` | auto-generated by Render |

   **`zero-hunger-web`**
   | Variable | Value |
   |---|---|
   | `VITE_API_BASE_URL` | the API URL **with `/api`** (e.g. `https://zero-hunger-api.onrender.com/api`) |

3. Because the two URLs reference each other, after the first build:
   - set `FRONTEND_URL` / `CORS_ORIGINS` on the API to the real web URL,
   - set `VITE_API_BASE_URL` on the web service to the real API URL,
   then trigger a redeploy of both.

## How each service builds

- **API:** `npm ci --include=dev && npm run build` (compiles TypeScript to
  `dist/`), then `npm start` runs `node dist/server.js`. Health check: `GET /health`.
- **Web:** `npm ci && npm run build` produces `dist/`, served as a static site
  with an SPA rewrite (`/* → /index.html`) and long-lived caching on `/assets/*`.

## Notes

- The API **refuses to boot** without a strong `JWT_SECRET` and a reachable
  `MONGO_URI` — this surfaces misconfiguration immediately in the deploy logs.
- Socket.IO runs on the same web service as the API (persistent connection),
  so real-time notifications work without extra configuration on a single instance.
- On Render's free tier the API may cold-start after inactivity; the first
  request can be slow.

## Local development

```bash
# Backend
cd Backend && cp .env.example .env   # fill in values
npm install && npm run dev           # http://localhost:5000

# Frontend
cd Frontend && cp .env.example .env  # VITE_API_BASE_URL=http://localhost:5000/api
npm install && npm run dev           # http://localhost:5173
```

> macOS note: port 5000 is often taken by AirPlay Receiver. Disable it
> (System Settings → General → AirDrop & Handoff) or set `PORT` to something else.
